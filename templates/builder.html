How it works

  • Every loaded pipeline module is listed below. Phases run top-to-bottom; a phase is either one stage or a group of stages running in parallel.
  • The connectivity-check gate splits each pipeline: stages above it run pre-gate (typically credential / association work), stages below run post-gate (probe / tunnel / bypass strategies).
  • If a pipeline has no connectivity_check phase it will always fail to confirm internet — add one from the palette.
  • Click the parallel toggle on a phase to fold it into the previous one, so they race; the fastest with real internet wins.
  • Changes save to MongoDB and apply to the running pipeline immediately. Reset returns to the module's hard-coded defaults.
