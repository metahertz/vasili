# vasili-helper

A single Docker image bundling all the server-side counterparts of vasili's
network-bypass modules, with a small web UI to configure them.

> **New here?** Start with the repo-root [`README.md`](../README.md) for the
> end-to-end "spin up server, then point the Pi at it" flow. This file is
> the helper's own reference once you're working on the server-side.

## TL;DR

```bash
# On a VM with a public IP, UDP/53 and TCP/53 free, Docker installed:
docker compose -f helper/docker-compose.yml up -d --build
docker logs vasili-helper | grep HELPER_TOKEN
```

Open `http://<vm-ip>:8080`, paste the token, toggle the services you want
(any combination of SSH-on-53, iodine, WireGuard, crack server), follow
the DNS delegation hints the UI shows for each domain you configure,
then paste the **Client Config** block into the matching stages on the
vasili Pi. All UDP/53 services can run at the same time on a single
public IP — the proxy demuxes by packet shape.

What's inside:

| Service | Port | Purpose |
|---|---|---|
| Helper UI (Flask) | 8080 | Configures everything below |
| DNS proxy / UDP/53 mux | 53/udp | Classifies each packet: WireGuard-shaped → wg-backend; DNS with `crack-domain` suffix → crack server; DNS with `iodine-domain` suffix → iodine; else drop |
| PMKID crack server | 5353/udp (internal) | Receives DNS-encoded PMKID hashes, runs hashcat |
| iodine | 5354/udp (internal, fronted on 53/udp) | DNS-over-DNS tunnel |
| WireGuard | 5355/udp (internal, fronted on 53/udp) | Binary VPN tunnel, demuxed by packet shape |
| sshd (TCP/53) | 53/tcp | SSH tun-mode VPN for the `dns_port_tunnel` stage |

**All three UDP/53 services can run simultaneously** on a single public IP
— the proxy demuxes by packet signature (WireGuard) and DNS qname suffix
(iodine vs crack). The only constraint is that `iodine_domain` and
`crack_domain` must be disjoint (neither a suffix of the other); the UI
rejects overlapping configurations.

The existing `server/vasili-crack-server.py` and `server/vasili-dns-proxy.py`
are dropped in unmodified — the helper just writes their JSON configs and
restarts them via `supervisorctl`. The original `vasili-server-setup.sh`
bash menu still works for non-Docker installs.

## Quickstart

```bash
# from the repo root
HELPER_TOKEN=$(openssl rand -hex 16)
echo "Token: $HELPER_TOKEN"
docker compose -f helper/docker-compose.yml up -d --build
```

Browse to `http://<host>:8080`, paste the token, and configure each
section. The UI shows the exact DNS NS/A records to set at your registrar
and emits a copy-paste block of vasili-side config for each enabled
service.

## Required ports on the host

- UDP/53 and TCP/53 must be free. If `systemd-resolved` is bound to :53,
  add a drop-in:
  ```
  # /etc/systemd/resolved.conf.d/no-stub.conf
  [Resolve]
  DNSStubListener=no
  ```
  then `systemctl restart systemd-resolved`.
- TCP/8080 is the UI; firewall it or front it with a reverse proxy.

## Authentication

The UI uses bearer-token auth. Set `HELPER_TOKEN` in the environment to
pin a token; otherwise one is generated on first boot and printed to
`docker logs`. The token also lives in `/etc/vasili-helper/config.json`
inside the named volume.

## Wordlist

`rockyou.txt` is **not** bundled (license + size). Click "Download
rockyou.txt" in the Crack section to fetch ~134 MB into the
`helper-state` volume.

## Files copied to vasili (Pi side)

After enabling services, the UI exposes downloads:

- **SSH client key** → place at `/etc/vasili/ssh_client_key` on the Pi
  and point the `dns_port_tunnel` stage at it.
- **WireGuard client config** → place at
  `/etc/wireguard/wg-vasili-client.conf` on the Pi and point the
  `dns_port_tunnel` stage at it.
- **Crack server creds** → drop the printed `offload_domain` /
  `offload_secret` into the `dns_offload_crack` stage config.
- **Iodine creds** → drop the printed `server_domain` /
  `tunnel_password` into the `dns_tunnel` stage config.

## DNS delegation (mandatory for iodine / crack)

For `crack.example.com` and `t.example.com` to receive UDP/53 traffic,
add at your registrar/DNS provider:

```
crack.example.com.    NS   ns-crack.example.com.
ns-crack.example.com. A    <PUBLIC_IP_OF_HELPER>

t.example.com.        NS   ns-vasili.example.com.
ns-vasili.example.com. A   <PUBLIC_IP_OF_HELPER>
```

The "DNS delegation" section in the UI fills these in for you using your
configured domains and detected public IP.

## Notes

- WireGuard's listener is configured for **5354/udp internal** so the DNS
  proxy can sit in front on 53/udp. When iodine is the chosen backend, it
  binds 5354 the same way.
- `wg-vasili.conf` and `vasili-sshd-53.conf` are generated by the
  launcher shims from `config.json` on every (re)start; don't edit them
  by hand.
- The container runs everything as root. The image isn't designed to be
  exposed unauthenticated — keep 8080 behind a firewall, Tailscale, or
  reverse proxy.

## Verification

```bash
# 1. Build
docker compose -f helper/docker-compose.yml build

# 2. UI smoke (no privileged ports needed)
docker run --rm -p 8080:8080 -e HELPER_TOKEN=dev vasili-helper:latest
# Visit http://localhost:8080 — paste "dev" — confirm everything shows STOPPED

# 3. Full run (needs free 53/udp + 53/tcp)
docker compose -f helper/docker-compose.yml up -d
# Configure crack-server domain=crack.test, secret=foo. Apply.
dig @127.0.0.1 -p 53 \
    aabbccddeeff.001122334455.001122334455.74657374.submit.foo.crack.test
# Expect: A 1.0.0.1
```
